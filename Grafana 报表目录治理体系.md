# 📊 Grafana 报表治理规范 (v1.0)

## Grafana 报表目录治理体系

```text
Grafana 报表目录治理体系
│
├── 📝 报表命名规范 
│   │   ℹ️ 适用于以下所有目录内的报表命名，确保列表排序清晰
│   │
│   │
│   ├── 场景 A：上报明细 / 原始日志
│   │   └── 命名：{业务}-前端/后端 - {数据名}
│   │       └── 例：直播-前端-SDK用户心跳
│   │       └── 例：直播-后端-转码任务情况
│   │       └── 例：AI-后端-AIGC使用明细
│   │  
│   ├── 场景 B：单一指标 / 核心KPI (专注单个指标)
│   │   └── 命名：{业务} - {主指标/场景} - {细分范围,可选} - {主维度,可选} - {时间粒度}
│   │       └── 例：直播 - 卡顿排查 - SDK模块 - 分主播 - 分时点
│   │       └── 例：直播 - 王者陪练成本 - 按天
│   │
│   └── 场景 C：多指标聚合 / 综合分析 (大盘/排查看板)
│       └── 命名：业务/全网-场景聚合 - [细分范围] - 综合分析 - [时间粒度]
│           └── 例：全网-版本质量 - 综合分析 - 按天
│           └── 例：直播-CDN日志带宽 - 综合分析 - 峰值点
│
│
└── 📝 目录命名规范
    │ 
    ├── 📂 00. 核心报表  🚀
    │   ├── 定位：整体/跨业务线核心KPI一览；负责人专用，只看结果，不查明细
    │   ├── 命名：虎牙/NIMO业务核心报表
    │   └── 注意：严格控制该目录内报表数量，一般报表不放入该目录
    │
    ├── AI类项目 [项目制：单独产品闭环] 🤖
    │   │
    │   ├── 命名规范：标准目录名 = AI-{项目名}
    │   │
    │   └── 按指标大类划分
    │       ├── 质量 → AI-{项目名}-质量
    │       │   （延时、准确率、连通性）
    │       ├── 成本/用量 → AI-{项目名}-成本/用量
    │       │   （Token消耗、带宽、API次数）
    │       ├── 技术排查 → AI-{项目名}-技术排查与明细 
    │       │   （全链路排查、过程指标、成功率、失败率、DEMO数据等）
    │       └── 业务数据 → AI-{项目名}-业务数据
    │           （DAU、活跃度、业务转化）
    │
    ├─── 音视频项目 [业务制：按业务线划分] 📹
    │    │
    │    ├── 业务线 
    │    │   ├── 直播
    │    │   ├── 点播
    │    │   ├── 静态资源
    │    │   ├── 弹幕
    │    │   └── XXX（新业务，例如：云游戏、爆爆语音等）
    │    │
    │    │
    │    └── 二级结构 (适用于上述任一业务线)
    │        ├── 类型 1：上报明细 (存放原始数据及上报质量)
    │        │   ├── 命名逻辑：{业务线}-上报明细-{前端/后端}-{模块}
    │        │   ├── 前端模块：用户/主播 (面向使用者)
    │        │   └── 后端模块：P2P/转码/媒体加工等 (后台处理模块)
    │        │
    │        └── 类型 2：统计数据 (加工指标，KPI) 📊
    │            │
    │            ├── 核心分类逻辑：按使用场景
    │            │   ├── 核心指标展示 (KPI、宏观、向上汇报)
    │            │   ├── 综合分析 (多指标、排查、常用监控)
    │            │   └── 专项分析 (临时、细分模块、竞品、实验)
    │            │
    │            └── 具体业务域落地规则
    │                │
    │                ├── 前端质量 (卡顿/黑屏/秒开/延时，数据源为SDK/内部上报)
    │                │   ├── 核心指标 → {业务}-卡顿-卡顿率
    │                │   ├── 综合分析 → {业务}-卡顿-综合分析 (原因/过程)
    │                │   └── 专项分析 → {业务}-卡顿-{模块}排查 (SDK/传输)
    │                │
    │                ├── 后端模块（P2P、转码、PCDN等，数据源为SDK/内部上报） 
    │                │   ├── 核心指标 → {业务}-P2P-P2P分享率
    │                │   ├── 综合分析 → {业务}-P2P-P2P综合分析 (流量/质量/节点)
    │                │   └── 专项分析 → {业务}-P2P-P2P专项分析 (竞品/实验)
    │                │
    │                ├── CDN带宽与成本 (数据源区分 SDK vs CDN) 💰
    │                │   │
    │                │   ├── 判定 1：是否涉及具体模块/项目？(如转码、P2P、PCDN)
    │                │   │   └── 是 → 归类：专项分析
    │                │   │       └── 命名：{业务}-{模块}-{来源}带宽专项分析
    │                │   │           (例：直播-转码-sdk带宽专项分析)
    │                │   │
    │                │   └── 判定 2：无具体模块 (通用带宽)
    │                │       └── 归类：综合分析 / 成本
    │                │           ├── SDK 侧 → {业务}-sdk带宽-综合分析
    │                │           ├── CDN 侧 → {业务}-cdn日志带宽-综合分析
    │                │           └── 全站/成本 → 媒体-cdn日志带宽-成本
    │                │               (例：媒体-cdn日志带宽-带宽用量)
    │                └── IDC成本
    │                    │
    │                    ├── 判定 1：本部门IDC成本 (内部成本核算)
    │                    │    │    
    │                    │    └── 命名：媒体-智能媒体IDC成本-综合分析
    │                    │        (监控部门整体IDC成本：算力占用、存储、网络、内部接口调用等)
    │                    └── 判定 2：提供对外服务的IDC成本 (监控核算对应服务/项目的成本 ，例如Leaf)
    │                        │   
    │                        └── 命名：媒体-Leaf IDC成本-综合分析
    │                              (监控对应服务 全公司调用量、分摊成本等)
    │
    │
    ├── 📂  整体/跨业务线聚合类报表
    │   │       
    │   ├── 定位：整体/跨业务线的聚合报表，包含多类指标，无法归类到 AI类数据体系和音视频数据体系
    │   └── 命名：根据功能点命名，例如 ：全网-成本-综合分析
    │
    │
    └─── 📂 其他非业务数据 [数据治理类报表/告警系统报表/报表开发模板等]
        │       
        ├── 定位：特定功能的报表，用于数据服务
        └── 命名：根据功能点命名，例如 ：告警系统 、报表开发模板等
