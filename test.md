# 📊 Grafana 报表治理规范 (v1.0)

> **文档维护人**: 数据治理组
> **最后更新**: 2023-10-27
> **适用范围**: 全公司 Grafana 报表创建与维护

## 1. 核心原则
*   **业务隔离**：严格区分 AI 项目与音视频业务。
*   **功能分离**：核心 KPI、告警、开发模板需独立存放。
*   **命名规范**：所有 Dashboard 必须遵循下文定义的命名逻辑，违规报表将被定期清理。

## 2. 目录结构总览 (Directory Structure)

```text
Grafana 报表治理体系
│   
├── 📂 00. 核心报表  🚀
│   ├── 定位：整体/跨业务线核心KPI一览；负责人专用，只看结果，不查明细
│   ├── 命名：虎牙/NIMO业务核心报表
│   └── 注意：严格控制该目录内报表数量，一般报表不放入该目录
│
├── AI类项目 [项目制：单独产品闭环] 🤖
│   │
│   ├── 命名规范：标准目录名 = AI-{项目名}
│   │
│   └── 分类维度 (Dashboard 命名)
│       ├── 质量 → AI-{项目名}-质量
│       ├── 成本/用量 → AI-{项目名}-成本/用量
│       ├── 业务用量 → AI-{项目名}-业务用量数据
│       └── 技术排查 → AI-{项目名}-技术排查与明细
│
├─── 音视频项目 [模块制：直播/点播/静态/弹幕/xxx] 📹
│    │
│    ├── 类型 1：上报明细 (存放原始数据及上报质量)
│    │   ├── 命名逻辑：{业务线}-上报明细-{前端/后端}-{模块}
│    │   ├── 前端模块：用户/主播 (面向使用者)
│    │   └── 后端模块：P2P/转码/媒体加工等 (后台处理模块)
│    │
│    └── 类型 2：统计数据 (加工指标，KPI) 📊
│        │
│        ├── 核心分类逻辑：按使用场景 (核心/综合/专项)
│        │
│        └── 具体业务域落地规则
│            ├── 前端质量 (数据源: SDK)
│            │   ├── 核心指标 → {业务}-卡顿-卡顿率
│            │   ├── 综合分析 → {业务}-卡顿-综合分析
│            │   └── 专项分析 → {业务}-卡顿-{模块}排查
│            │
│            ├── 后端模块 (数据源: P2P/转码)
│            │   ├── 核心指标 → {业务}-P2P-P2P分享率
│            │   └── 专项分析 → {业务}-P2P-P2P专项分析
│            │
│            ├── CDN带宽与成本 💰
│            │   ├── 专项分析 → {业务}-{模块}-{来源}带宽专项分析 (如转码)
│            │   ├── 综合分析(SDK侧) → {业务}-sdk带宽-综合分析
│            │   └── 成本核算(全站) → 媒体-cdn日志带宽-成本
│            │
│            └── IDC成本
│                ├── 部门自用 → 媒体-智能媒体IDC成本-综合分析
│                └── 对外服务 → 媒体-Leaf IDC成本-综合分析
│                       
├── 📂 整体/跨业务线聚合类报表                            
│   ├── 定位：无法归类到单一业务线的聚合报表
│   └── 命名：全网-成本-综合分析                   
│                          
└── 📂 其他非业务数据 [数据治理/告警/模板]                            
    ├── 定位：特定功能的报表，用于数据服务
    └── 命名：告警系统 / 报表开发模板
